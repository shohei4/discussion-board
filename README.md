# 議論アプリ
議論アプリケーションをSpringで改良
# 作成計画
行程
## step1: Singleton + Factory Method
**目的**
・Bean管理(Sigleton)と生成責務分離(Factory Method)の理解
・投稿とユーザーのインスタンス生成をFactoryクラスにまとめる
**期待できる成果**
・Factory Method + Singletonパターンの理解
・SpringのBeanスコープの管理
## step2: Abstract Factory
**目的**
・APIレスポンス形式(DTO)をまとめて生成する
・例）投稿＋ユーザー情報付きレスポンスを一括生成
**期待できる成果**
・一貫したレスポンス設計
・抽象層と具象層の分離（拡張性）
## step3: Template Method
**目的**
・投稿保存の処理フローを共通化、差分のみをサブクラスで実装
・例）通常、管理者投稿や下書き投稿の共通化
**期待できる成果**
・一定手順の共通化
・OCP（開放閉鎖原則）に近い構造理解
## step4: Strategy
**目的**
・投稿一覧の並び替え/フィルタロジックを切り替えられる設計
**期待できる成果**
・条件分岐ではなくオブジェクト切り替えで柔軟性を実現
・戦略切り替えの概念理解（OCP/DI応用）
## step5: Observer
**目的**
・投稿やいいね時に「イベント」を発火し、別クラスで処理を受け取る
**期待できる成果**
・「処理の分離」「非同期的イベント処理」の理解
・OOP＋SpringのObserver構造体験
## step6: 統合・リファクタリング
**目的**
・各パターンを整理して、拡張しやすいアプリに仕上げる
・学習成果の確認
**期待できる成果**
・OOP設計の流れが身につく
・Springでのデザインパターン実装ポートフォリオが完成
