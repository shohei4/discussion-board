# 議論アプリ
* 議論アプリケーションをSpringで改良
## アプリの目的
* ユーザー同士で価値観や意見の交換を行うことで、知見を広げる機会を設けることを目的としている。

## 開発期間
* 約3か月（2025年10月下旬から2026年1月中旬）

## 開発環境
* Windows 10
* Eclipse 2023-12
* Java SE-21
* PostgreSQL 17

## アプリの機能一覧
* ログイン・ログアウト機能
* 議題の投稿・更新、一覧表示
* 議論コメントの投稿、一覧表示
* 議論コメントへのいいね機能

## 工夫した点
**認証機能**
* アクセストークンのリフレッシュトークンを使用して、ステートレスに認証する形式にした。
* リフレッシュトークンに使用フラグを持たせることで、第三者による悪用を検知しやすい設計にした。
<br/>
**設計**
<br/>
* 認証処理を回避する対象URLや例外のコード・メッセージ・リダイレクト先などを、Enumを利用して扱いやすくした。
* Rest設計に挑戦した。
* DTOの使用により、情報のカプセル化を行った。
* 例外処理を独自クラスでラッピングすることで、ControllerAdvice＋ExceptionHandlerで一括ハンドリングできるようにした
* フロント側の認証処理を関数化して、コード量を減らし、開発コストを下げ、可読性を上げた。
* viewとapiのコントローラーを分けて、認証処理をapiに一元化する、責務の分離などを意識した設計にした。
* バックエンド側の責務の分離を意識して設計した。具体的には、ControllerはServiceメソッドでの呼び出しに従事させ、条件分岐やロジックはService内で完結するようにした。
